## Objective
The objective of this project is to implement palindrome checking using an object-oriented design.
By encapsulating the logic inside a dedicated service class, the program becomes more modular, reusable, and easier to extend.

## Flow of the Project
1. 	Input Handling
      • 	User enters a string.
      • 	The string is passed to the palindrome service.
2. 	Service Class (PalindromeService)
      • 	Normalizes the string (removes spaces, converts to lowercase).
      • 	Uses a two-pointer approach to check if the string is a palindrome.
3. 	Main Class (UseCase11)
      • 	Creates an instance of the service.
      • 	Calls the  method.
      • 	Displays the result.

## Use Case
• 	Educational Purpose: Demonstrates object-oriented principles like encapsulation and separation of concerns.
• 	Reusability: The palindrome service can be reused in other projects or extended with new features (e.g., ignoring punctuation).
• 	Interview Preparation: Shows understanding of both data structures and object-oriented design.
• 	Real-world Applications:
• 	Can be integrated into larger text-processing systems.
• 	Useful in modular applications where palindrome checking is one of many services.
